🖨️ Epson Printer Controller API

REST API for Epson TM-T20III thermal printer control

Endpoints

GET /api/printer/status

Get current printer status including paper level, cover state, and errors

GET /api/printer/health

Health check endpoint - returns 200 if printer is ready, 503 if not

POST /api/printer/print

Print structured content with formatting options (JSON)

{
  "text": "Hello World",
  "items": [
    {
      "type": "TEXT",
      "content": "Sample text",
      "options": {
        "bold": true,
        "alignment": "CENTER",
        "fontSize": 2
      }
    },
    {
      "type": "IMAGE",
      "content": "base64_or_file_path",
      "options": {
        "width": 200
      }
    },
    {
      "type": "BARCODE",
      "content": "123456789"
    },
    {
      "type": "LINE",
      "content": "================================"
    }
  ],
  "cutPaper": true,
  "copies": 1
}
POST /api/printer/print/text

Print plain text (send text as raw body)

curl -X POST -d "Hello World!" http://localhost:8080/api/printer/print/text
POST /api/printer/cut

Cut paper (if printer supports paper cutting)

POST /api/printer/test

Test print functionality - prints a test page

GET /api/events/status

Server-Sent Events for real-time printer status updates

Status Monitoring

Connect to /api/events/status to receive real-time printer status updates via Server-Sent Events.

Status updates include:

  • Online/offline state
  • Paper empty/near end warnings
  • Cover open detection
  • Error conditions

Example Usage

// Get status
curl http://localhost:8080/api/printer/status

// Print text
curl -X POST -d "Hello World!" http://localhost:8080/api/printer/print/text

// Print structured content
curl -X POST -H "Content-Type: application/json" \
  -d '{"items":[{"type":"TEXT","content":"Test","options":{"bold":true}}],"cutPaper":true}' \
  http://localhost:8080/api/printer/print

// Test print
curl -X POST http://localhost:8080/api/printer/test

// Health check
curl http://localhost:8080/api/printer/health

// Listen to status updates (JavaScript)
const eventSource = new EventSource('/api/events/status');
eventSource.onmessage = function(event) {
    const status = JSON.parse(event.data);
    console.log('Printer status:', status);

    if (status.paperEmpty) {
        alert('Paper is empty!');
    }
    if (status.coverOpen) {
        alert('Cover is open!');
    }
};

// Close SSE connection when done
eventSource.close();

Response Format

All endpoints return JSON responses with the following structure:

{
  "success": true,
  "message": "Operation completed successfully",
  "data": { /* response data */ },
  "timestamp": 1234567890
}

Error Handling

Errors return appropriate HTTP status codes:

  • 400 - Bad request (invalid input)
  • 500 - Internal server error (printer error)
  • 503 - Service unavailable (printer offline)

Error response format:

{
  "success": false,
  "message": "Error description",
  "data": null,
  "timestamp": 1234567890
}
